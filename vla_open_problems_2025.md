## 1. 合成一般化と長期タスク

### 1.1 何が問題か

VLAにおける合成一般化（compositional generalization）とは、単発タスクの成功率ではなく、**既習の原子スキルを新しい順序・新しい組合せで"タスク列"として構成し、外乱の中で破綻なく遂行する**こと。これは多くの研究者が「基盤モデル化の本丸」と捉えている。

LIBERO-PROが丸暗記を暴いた（→ vla_knowledge_base.md セクション7.1）のに加え、**ClevrSkills**が問題をさらに具体化した:

| 項目 | 内容 |
|------|------|
| **論文** | ClevrSkills |
| **時期** | 2024-11 |
| **環境** | ManiSkill2 上の 33 タスク、33万軌跡 |
| **設計** | 3段階の合成理解レベル（原子スキル → 2スキル合成 → 多スキル合成） |
| **知見** | **多タスクで事前学習しても、合成推論を要するロボティクス課題で失敗する** |
| **意義** | 「計画と実行を閉ループで同時に問う」ベンチマークの登場自体が、現状の弱点を可視化 |

### 1.2 有望な解の方向性

2024年後半〜2025年初頭の"有望解"は、単なるモデルサイズ拡大ではなく、**中間表現を入れて合成・推論の構造を強制する**方向に集中している。3つの流れがある。

#### A. Chain-of-Thought の身体化

通常の CoT プロンプトは VLA では効きにくい（訓練例が単純で、純テキスト推論が観測・ロボ状態に接地しにくい）。これを解決する 2 つのアプローチ:

**ECoT（Embodied Chain-of-Thought）**

| 項目 | 内容 |
|------|------|
| 手法 | 計画・サブタスク・運動・バウンディングボックスや EE 位置を含む推論ステップを出力してから行動を決めるよう学習 |
| 結果 | OpenVLA の成功率を追加ロボデータなしで大きく引き上げ |
| 副次効果 | 推論チェーンを点検して「どこで誤認識したか」を人が見つけ、自然言語で修正できる（**介入可能性**） |

**CoT-VLA（2025-03）**

| 項目 | 内容 |
|------|------|
| 問題意識 | 従来 VLA は入出力写像に寄りがちで「時間的な計画・推論が欠ける」 |
| 手法 | 将来フレームを自己回帰で予測して**"視覚ゴール列"**を作り、その後に短いアクション列を生成 |
| 結果 | 実機・シミュレーション双方で性能向上を報告 |
| 意義 | **「予測＝推論＝制御」の統合**。ワールドモデル的要素を VLA 推論の中間ステップへ組み込む |

> **既存ナレッジベースとの関係**: vla_knowledge_base.md セクション 8.4「推論の導入」で CoT-VLA / CoA-VLA / EF-VLA を簡潔に記載済み。ここでは ECoT を追加し、「なぜ通常 CoT が効かないか」の理由を補足。

#### B. 階層化によるスキル合成

**HAMSTER（2025-02）**

| 項目 | 内容 |
|------|------|
| 問題意識 | 安価なオフドメインデータ（行動なし動画・スケッチ・シミュレーション等）を活用したい |
| アーキテクチャ | 高レベル VLM が**粗い 2D パス**（EE の軌道ガイド）を予測 → 低レベルの 3D 制御ポリシーが精密操作を担う |
| 利点 | 高レベル側は意味・視覚ギャップに強く、低レベル側は幾何と操作の頑健性に集中 |
| 結果 | OpenVLA に対して複数の一般化軸で成功率改善 |

**π0 の階層設計**

π0 も「高レベルの意味的ポリシーが中間言語コマンドを出し、低レベル VLA が実行する」階層を明示しており、長期タスクへ向けた"スキル列の構成"を射程に入れている。

#### C. 合成一般化のまとめ

| 流れ | 代表 | 核心 | 既存 KB との関係 |
|------|------|------|-----------------|
| テキスト CoT の身体化 | ECoT | 推論ステップ + 接地 + 介入可能性 | セクション 8.4 に追加 |
| 視覚 CoT | CoT-VLA | 未来フレーム予測→行動 | セクション 8.4 を深掘り |
| 階層化 | HAMSTER, π0 | 意味(高)×幾何(低)の分離 | **新規** |
| ベンチマーク | ClevrSkills | 3段階の合成理解レベル | **新規** |

### 1.3 まだ未解決なこと

- **新規組合せへの汎化**: 既習スキルの未見の順列組合せは依然として崩壊しやすい
- **長期記憶**: エピソード長が伸びるとコンテキスト管理が困難
- **外乱下での閉ループ回復**: 計画通りに行かなかったときの再計画能力が弱い

## 2. Sim-to-Real と実機展開

### 2.1 現状の立ち位置

2025年初頭の最先端 VLA の多くは、**中核部分を実ロボットデモで学習している**。純粋な "sim-only で訓練→実機にゼロショット投入" は、まだ限定的。

- OpenVLA: 実ロボデモ **97万軌跡** で学習
- OXE: 実ロボ軌跡を核にしたクロスエンボディメント学習の基盤
- π0: 7つの実ロボットプラットフォームの自社データ + OXE

### 2.2 3つのギャップ

コミュニティの実務的焦点は、訓練の sim-to-real だけでなく「**評価・デバッグを現実寄りにする real-to-sim**」にも強く向いている。

**SIMPLER** がギャップを以下の 3 つに分解した:

| ギャップ | 内容 | 対策 |
|---------|------|------|
| **見た目（Visual）** | 撮像・照明・背景・テクスチャに敏感。評価すら信頼できない | グリーンスクリーン + テクスチャ整合（Visual Matching） |
| **制御（Control）** | 多様なロボット・多様な制御ループを一律に扱う困難 | SysID（システム同定）で埋める |
| **レイテンシ** | 推論の遅さが「ダイナミクスのズレ」として顕在化 | → セクション 3 で詳述 |

SIMPLER の具体的な知見:
- 実環境と相関するシミュレーション評価を構築
- 分布シフト感度（照明、背景、カメラ姿勢、テーブルテクスチャ等）に対するポリシー感度が実機と強く相関
- Google Robot は 3Hz、WidowX は 5Hz で評価

### 2.3 生成モデルで sim-to-real を縮める

**UniSim**

| 項目 | 内容 |
|------|------|
| コンセプト | 「実世界相互作用の普遍シミュレータ」を生成モデルとして学習 |
| 手法 | 異種データセットをオーケストレーションし、言語指示や低レベル制御に応じた視覚結果をシミュレート |
| 主張 | シミュレータ上で学んだ方策が**純シミュレーション学習からのゼロショット実機展開**を可能にする |

### 2.4 レイテンシがダイナミクスを変える問題

OpenVLA が重要な因果関係を示唆:

```
8-bit量子化 → 推論が遅くなる → 制御周波数が下がる
    → データ収集時のダイナミクスと合わない → 性能低下
```

- ブロッキング制御（次アクション算出まで前アクションを保持）で影響を除くと差が縮む、という検証あり
- **「sim-to-real 以前に、train-time と test-time の制御ループ整合が必要」** という論点

### 2.5 2025年初頭の結論

主戦場は「高忠実度の sim を作る」よりも:

1. **実データを中核にしつつ**
2. **評価や安全学習のために sim / 生成モデルを賢く使い**
3. **見た目・制御・レイテンシを分解して詰める**

方向へ寄っている。

## 3. リアルタイム推論と計算効率

### 3.1 ボトルネックの具体値

| モデル | メモリ | 推論速度 | 備考 |
|--------|--------|---------|------|
| OpenVLA (7B, bf16) | ~15 GB | ~6 Hz（RTX 4090） | 追加の高速化なし |
| π0 (3B + Action Expert) | — | 最大 50 Hz | Flow Matching + Action Chunk |
| ALOHA タスク | — | 50 Hz 要求 | 高頻度双腕操作 |

**「7B級を10Hz以上で回す」は 2025年初頭で自然に達成できる前提ではない。**

### 3.2 5つの高速化カテゴリ

#### A. 小型化・高速化モデル設計

**TinyVLA**

| 項目 | 内容 |
|------|------|
| 問題意識 | 「推論が遅い / 巨大な事前学習データが必要」 |
| 手法 | (1) 高速マルチモーダルモデルでバックボーン初期化 (2) 微調整時に拡散ポリシーデコーダを統合 |
| 主張 | 高速推論とデータ効率の両立。実機評価あり |

#### B. 動的計算（Early Exit）

**DeeR-VLA（2024-11）**

| 項目 | 内容 |
|------|------|
| 手法 | マルチ出口構造により、入力の難しさに応じて早期終了 |
| 結果 | CALVIN で LLM 計算を **5.2〜6.5倍削減**、LLM の GPU メモリを **2〜6倍削減**、性能維持 |
| 意義 | 「簡単な入力に大きなモデルは不要」という直感を実装 |

#### C. キャッシュと再利用

**VLA-Cache（2025-02）**

| 項目 | 内容 |
|------|------|
| 着眼点 | 連続フレーム間で視覚入力がほとんど変わらない |
| 手法 | 変化の小さい視覚トークンの KV を再利用して推論を省略 |
| 結果 | CUDA レイテンシ **最大1.7倍改善**、制御周波数 **15%向上** |
| 検証 | SIMPLER + 実機でも効果を確認 |

#### D. 量子化・圧縮

| 手法 | 内容 |
|------|------|
| **OpenVLA の 4-bit 量子化** | bf16 と同等性能、GPU メモリ半分以下。ただし制御周波数とダイナミクス整合に注意 |
| **Quantization-Aware Imitation Learning（2024-12）** | 4-bit 量子化に対する頑健性を訓練中に高める。エッジ GPU で **最大2.5倍の速度・省電力改善** |

#### E. 行動表現・デコーディングの再設計

| 手法 | 内容 |
|------|------|
| **FAST / FAST+（2025-01）** | DCT 圧縮型トークナイザ。汎用トークナイザ FAST+ を公開。π0 と組み合わせると拡散 VLA と同等性能で**学習時間を最大5倍短縮** |
| **OFT / OpenVLA-OFT（2025-02）** | 並列デコーディング + アクションチャンク + 連続行動表現 + L1 回帰で、行動生成スループット **26倍**。LIBERO で成功率も大幅向上 |
| **Speculative Decoding** | 軽量ドラフトモデルで提案 → 本体が検証する並列化。VLA のアクショントークン生成にも概念的に適用可能 |

### 3.3 高速化手法の比較

| カテゴリ | 代表 | 削減幅 | 性能への影響 | 実装難易度 |
|---------|------|--------|------------|----------|
| 小型化 | TinyVLA | モデルサイズ削減 | △ 能力制限あり | 中（モデル再設計） |
| Early Exit | DeeR-VLA | 計算 5〜6倍削減 | ○ 性能維持 | 中（出口追加） |
| KV Cache | VLA-Cache | レイテンシ 1.7倍改善 | ◎ ほぼ無損失 | **低（推論時のみ）** |
| 量子化 | 4-bit QAT | メモリ半分、速度 2.5倍 | ○ 注意が必要 | 低〜中 |
| 行動表現刷新 | OFT | スループット 26倍 | ◎ 成功率も向上 | 中（学習パイプ変更） |
| Speculative | 概念段階 | 理論的に数倍 | ? 未検証 | 高 |

## 4. 安全性と信頼性

### 4.1 なぜ難しいか

VLA の安全性は「成功率が低い」とは質的に異なる。失敗の仕方が物理世界で**高コスト（破損・危険・不可逆）**になりうる。

### 4.2 観測されている失敗モード

| 失敗モード | 観測元 | 具体例 |
|-----------|--------|--------|
| **分布シフト脆弱性** | SIMPLER | 照明、背景、カメラ姿勢、テーブルテクスチャ等で挙動が不安定化 |
| **誤認識→誤行動** | GR-2 | 未知形状の物体を把持できない、未知対象に対して誤った物体を選ぶ |
| **不安全行動** | SafeVLA | タスクと無関係な物体の安全を無視して危険物を動かす/損なう |
| **原因不明の失敗** | ECoT | 「なぜ失敗したか分からない」ため修正もできない |

### 4.3 3つの解決方向

2025年初頭の安全・信頼性の論点は、3つの方向に収束しつつある:

```
                    安全・信頼性の三角形

        安全制約を学習目的へ織り込む（SafeVLA）
                     ▲
                    / \
                   /   \
                  /     \
     分布シフト・失敗       中間推論を露出させ、
     モードを安全に         人が原因究明・修正
     再現して評価する        できるようにする
      （SIMPLER）          （ECoT等）
```

#### SafeVLA（2025-03）

| 項目 | 内容 |
|------|------|
| 問題意識 | VLA が物理世界と直接相互作用できることが安全問題を複雑化する |
| 手法 | **CMDP（制約付きMDP）** として安全制約（Object Safety / Robot Safety）を明示的に学習目的に組み込み |
| データ | 安全正例・負例を大規模に用いて学習 |
| 意義 | 安全を「後付けフィルタ」ではなく**学習時に統合**する方向の先駆 |

#### SIMPLER による安全な分布シフト評価

- シミュレーションで分布シフト感度を分析し、挙動モードが実機と対応することを確認
- 「安全な検証のためのシフト評価」を現実だけに頼らず回せる可能性

#### ECoT による予測可能性・介入可能性

- 推論チェーンを可視化 → 例: 工具の誤同定（ハンマーをドライバーと誤認）が誤行動に繋がる因果を追える
- **人が原因究明し、自然言語で修正できる**

## 5. ワールドモデル統合

### 5.1 なぜワールドモデルが必要か

VLA を「反射的な入出力写像」から押し出して、将来予測・"想像"を計画やデータ生成に結合する方向。背景には、**長期タスクでの合成一般化・分布シフト・データ効率**を、予測と生成で突破したいという動機がある。

### 5.2 4つのアプローチ

#### A. 動画予測 → 行動生成: GR-2（2024-10）

| 項目 | 内容 |
|------|------|
| 事前学習 | インターネット動画約 **3800万クリップ**（500億超トークン）で将来フレーム予測 |
| 微調整 | ロボット軌跡で「動画生成と行動予測を同時に微調整」 |
| 行動生成 | 軌道（trajectory）として生成 → 実機では Whole-Body Control で **200Hz** 追従 |
| 知見 | 動画生成が行動生成の**"プランナ"**として機能しうる（予測動画と実ロールアウトの整合が高い） |

#### B. 汎用シミュレータ: UniSim

| 項目 | 内容 |
|------|------|
| コンセプト | 異種データセットを組み合わせた生成モデリングで「汎用シミュレータ」を構築 |
| 対象 | 高レベル指示（「引き出しを開ける」）と低レベル制御の両方の視覚結果をシミュレート |
| 主張 | シミュレータで学んだ方策がゼロショットで実世界へ移れる |
| 意義 | **「sim-to-real 自体を生成モデルで縮める」** アプローチ |

#### C. 合成一般化をワールドモデルで解く: RoboDreamer

| 項目 | 内容 |
|------|------|
| 問題意識 | テキスト→動画生成モデルは学習時に見た指示に近い生成しかできず、**新規組合せに弱い** |
| 手法 | 言語の合成性に沿って指示をプリミティブへ分解し、動画生成を因子分解 |
| 結果 | **未見の組合せでも動画プランを生成**でき、RT-X 上で未見ゴールにも対応 |

#### D. デジタルツイン: DreMa（ICLR 2025）

| 項目 | 内容 |
|------|------|
| 問題意識 | 従来ワールドモデルが現実を直接・明示的に模倣できず、不自然な挙動や幻覚を生む |
| 手法 | **Gaussian Splatting + 物理シミュレータ** を統合して観測世界を複製 |
| データ生成 | 等価変換でデモから追加データを合成して模倣学習を強化 |
| 結果 | 実機 Franka で "1例/変種" に近い条件での学習を実証 |
| 意義 | データ効率と sim-to-real の両方に接続。**「学習可能デジタルツイン」** |

### 5.3 ワールドモデル統合のまとめ

| アプローチ | 代表 | 狙い | 成熟度 |
|-----------|------|------|--------|
| 動画予測→プランナ | GR-2 | Web動画の物理知識を活用 | 実機検証あり |
| 汎用シミュレータ | UniSim | sim-to-real を生成モデルで縮める | 概念実証段階 |
| 合成分解 | RoboDreamer | 未見組合せへの動画プラン生成 | シミュレーション検証 |
| デジタルツイン | DreMa | Gaussian Splatting + 物理でデータ生成 | 実機少数検証 |
| 視覚 CoT | CoT-VLA | ワールドモデルを推論の中間ステップに | 実機+sim 検証 |

## 主要論文マップ

2024年後半〜2025年初頭に、上記5つの問題軸を前に進めた代表論文を役割で整理。

### 基盤（インフラ・データ・モデル）

| 論文 | 時期 | 何が進んだか |
|------|------|------------|
| **OpenVLA** | 2024-06 | 実ロボデモ97万軌跡の7Bオープン VLA。LoRA・量子化・推論基盤まで配布可能な形で整備。推論 6Hz/15GB という現実的ボトルネックも具体化 |
| **OXE / RT-X** | ICRA 2024 | 22ロボット・21機関・527スキルの統合データ。クロスエンボディメント学習の標準土台 |

## まとめ: 解決されつつあること vs まだ未解決なこと

### 解決されつつあること

VLA 研究が "単発デモ模倣" から脱却し、**"システムとして詰める" 段階**に入った:

1. **データ基盤**: OXE による大規模・多ロボのクロスエンボディメント学習
2. **オープン VLA**: OpenVLA による再現可能な 7B 級 VLA
3. **高頻度連続制御**: π0 / FAST / OFT による行動表現の刷新
4. **推論高速化の体系**: Early Exit / KV Cache / 量子化の多角的アプローチ
5. **安全の学習統合**: SafeVLA による CMDP ベースの安全制約
6. **ワールドモデル統合**: GR-2 / CoT-VLA / UniSim / DreMa による予測と制御の結合

### まだ未解決なこと

| 問題 | 状況 |
|------|------|
| **合成一般化の本質** | 新規スキル列・長期記憶・外乱下での破綻回避が依然として難しい。強い VLM/VLA でもベンチマーク上で崩れる |
| **安全制約の実機導入** | 負例収集や sim-to-real が難しく、SafeVLA の枠組みを実運用に持ち込む道はまだ遠い |
| **推論レイテンシがダイナミクスを変える** | 7B級を高頻度制御へ適用する際、推論の遅さがダイナミクスそのものを変えてしまう現実的障害 |
| **VLM 知識の転移の条件** | どの種類の VLM 知識（意味/空間/因果/常識）が転移し、どれが転移しないか、の体系的理解がない |
| **評価の標準化** | 何をどう測るべきかの合意がない。LIBERO-PRO/Plus が方向性を示したが、まだ"デファクト"ではない |
