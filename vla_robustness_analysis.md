## 2. 既存ベンチマーク（LIBERO / LIBERO-PRO / LIBERO-Plus）の違い

### 2.1 設計思想の違い

| | LIBERO（元） | LIBERO-PRO | LIBERO-Plus |
|---|-------------|-----------|-------------|
| **目的** | マルチタスク汎化の測定 | **丸暗記の暴露** | **ロバスト性の体系的評価** |
| **学習とテストの関係** | **同一条件** | テスト時に摂動を注入 | テスト時に摂動を注入 |
| **摂動の数** | なし | 5軸 | 7軸 |
| **テストするモデル数** | — | 3（OpenVLA, π0, π0.5） | 10+ |
| **論文** | NeurIPS 2023 | arXiv:2510.03827 | arXiv:2510.13626 |

### 2.2 LIBERO（元）が測っていること

| スイート | タスク間で何が違うか | 各タスク内での学習-テスト条件 |
|---------|-------------------|--------------------------|
| LIBERO-Spatial | タスクごとに物体配置が異なる | **同一** |
| LIBERO-Object | タスクごとに物体種類が異なる | **同一** |
| LIBERO-Goal | タスクごとにゴール指示が異なる | **同一** |
| LIBERO-Long | 10ステップの連続タスク | **同一** |

各スイート: 10タスク × 50デモ。評価は学習時と**完全に同じ条件**で行うため、丸暗記で通ってしまう。

| 環境全体 | — | ○ | — |

### 2.4 Layout vs Position: 全く違う摂動

LIBERO-PRO の「Position」と LIBERO-Plus の「Layout」は名前が似ているが、**設計が根本的に異なる**。

| | LIBERO-PRO「Position」 | LIBERO-Plus「Layout」 |
|---|---|---|
| **動かす対象** | **ターゲット物体だけ** | **シーン内の全物体** |
| **動かす方向** | +x か +y の**単一軸、固定方向** | **ランダムな2D方向**（物体ごとに独立） |
| **動かす量** | 最大 **35cm**（7cm刻み × 5段階） | 最大 **約5cm** |
| **回転** | なし | ヨー回転も摂動 |
| **ランダム性** | 決定的（同じオフセットを全タスクに適用） | ランダム（1レベルにつき4サンプル） |
| **測っていること** | **ターゲットの絶対位置を暗記しているか** | **シーン全体の微変化に耐えるか** |

```
LIBERO-PRO（Position）:
  ターゲットだけを一方向にドカンとずらす

  学習時:  🥣 ← ターゲット(ボウル)    🍳フライパン   🫖ポット
  テスト:              🥣 ←── +14cm ──→  🍳           🫖
                    （他は動かない）

LIBERO-Plus（Layout）:
  全物体をそれぞれ少しずつランダムにずらす

  学習時:  🥣    🍳    🫖
  テスト:   🥣↗  🍳↙  🫖↘   （各±数cm、方向バラバラ）
```

このため結果も大きく異なる:

| モデル | LIBERO-PRO Position（14cm変位） | LIBERO-Plus Layout（最大5cm） |
|--------|-------------------------------|------------------------------|
| OpenVLA | **0%** | 31.6% |
| π0 | **0%** | 70.4% |

LIBERO-Plus の Layout は摂動量が小さいため、暗記した軌道でもギリギリ届く。LIBERO-PRO の方が「丸暗記」を暴く目的には鋭い。

## 3. カメラ摂動の詳細（LIBERO-Plus）

LIBERO の標準設定では、観測入力は **agentview（固定の三人称カメラ）** のみ。ハンドアイカメラ（robot0_eye_in_hand）は使っていない。カメラ摂動は全てこの固定カメラに対する変更。

### 3.1 3種類の操作

ピボット（回転中心）は `(0, 0, 0.8)` = テーブル面のほぼ中心。

#### C1: カメラ距離（313タスク）

ピボットからの距離を **1.15倍〜2.00倍** に拡大。カメラの向きは変えない。

```
  デフォルト位置 ●────── ピボット
  摂動後       ●──────────── ピボット（1.5倍遠い）
```

#### C2: 球面上の位置移動（992タスク、最も多い）

ピボットを中心とした球面上で、カメラ位置と向きを同時に回転。

```
  水平方向（方位角）: -75° 〜 +75°
  垂直方向（仰角）:   0° or 15°

       上から見た図:
            +75°
           ╱
      ピボット ─── 0°（デフォルト: 正面）
           ╲
            -75°
```

75°回り込むとシーンの見え方が根本的に変わる。最も壊滅的な摂動。

#### C3: カメラ姿勢のみ変更（294タスク）

カメラの位置は固定、向きだけ回転。

```
  ヨー（左右首振り）: ±2° 〜 ±10°
  ピッチ（上下首振り）: ±2° 〜 ±10°
```

最大10°だが、それでもモデルは崩壊する。

### 3.2 適用順序（コード上）

```
デフォルトカメラ位置・姿勢
  ↓ C2: 仰角回転（Y軸）
  ↓ C2: 方位角回転（Z軸）
  ↓ C1: 距離スケーリング
  ↓ C3: 姿勢ヨー回転
  ↓ C3: 姿勢ピッチ回転
  ↓
最終的なカメラ設定
```

### 3.3 なぜ固定カメラの微変更で壊滅するか

π0 が 94.2% → 15.8%（**-78pp**）に落ちる:
- VLA は画像の**ピクセル座標**と行動を直接対応づけて暗記している
- カメラが動くと、同じ物体が画像上の別の位置に映る
- **3D空間の理解がないことの直接的な証拠**
- 「カメラを三脚で設置し直したら動かなくなった」レベルの脆弱さ

π0-FAST だけ異常にカメラに強い（85.5% → 66.4%、-19pp）理由は未解明。FAST トークナイザの DCT 圧縮が視点変化に対してある種の不変性を持つ可能性がある。

## 5. 摂動軸の「効き具合」ランキング

全データを統合した総合評価:

```
壊滅的（ほぼ全モデルが崩壊）
  1. 物体の位置変更        → 14cmで 0%（LIBERO-PRO）
  2. タスク合成（A→B連続） → 全モデル 0〜1%（LIBERO-PRO）
  3. カメラ視点変更        → 最大 -78pp（LIBERO-Plus）
  4. ロボット初期姿勢      → 最大 -88pp（LIBERO-Plus）

中程度（モデルによって差が出る）
  5. 言語指示の変更        → -15〜-50pp（ただし理由が特殊※）
  6. レイアウト変更        → -15〜-45pp
  7. ノイズ（ブラー等）    → -10〜-57pp

比較的軽微（強いモデルなら耐える）
  8. 照明の変更            → -4〜-72pp（モデル差が極大）
  9. 背景テクスチャ        → -5〜-51pp（OFT系は -5pp 程度）
 10. 物体の色・外見        → -1〜-4pp（LIBERO-PRO）
```

※ 言語摂動の特殊性: パラフレーズしても成績が落ちない（OpenVLA: 0.98→0.98）。これは「ロバスト」ではなく**「そもそも言語を読んでいない」**から。無意味文字列「fdsgfdsgsd」に置き換えても同じ軌道を出力する。

## 6. ゼロショット能力の実態

### 6.1 「ゼロショットでできた」と主張している例

| モデル | 時期 | 何がゼロショットか | 成功率 | 信頼性 |
|--------|------|-------------------|--------|--------|
| **OTTER** | 2025-03 | 未見の物体に対する実機操作 | 62〜77%（OpenVLA は 0.6〜9%） | 非常に高い |
| **π0.5** | 2025-04 | 未見の家での片付け | 94%（OOD） | 高い（同一ロボット） |
| **GR-2** | 2024-10 | 未見の背景・環境・操作タスク | 背景71.4% / 環境71.7% / 操作55.8% | 高い（同一ロボット） |
| **RT-2** | 2023-07 | 推論を要する未見タスク | 62%（RT-1 の 32% → 約2倍） | 高い（emergent 能力） |
| **SpatialVLA** | 2025-01 | SimplerEnv でのゼロショット | 71.9%（RT-2-X 55B の 60.7% を 3.5B で超え） | 高い（sim 評価） |

### 6.2 重要な注意: 「新しいタスク」の意味

RT-2 の "emergent capabilities" の具体例:

| 指示 | 何が新しいか | 物理的な動作 |
|------|------------|------------|
| "Move banana to the sum of two plus one" | 算数推論 | **pick & place（既知）** |
| "Pick up the object that is different" | 差異推論 | **pick（既知）** |
| "Move coke can to Taylor Swift" | 有名人認識 | **pick & place（既知）** |

**「新しい」のは言語理解・推論の部分だけ**で、物理的な動作は全て学習済みスキル（pick, place, move）の範囲内。

```
VLA がゼロショットで "できる" こと:
  学習済みの動作スキル × 未見の物体/場面/指示の組み合わせ
  （VLM の知識で "何を" の解釈が広がる）

VLA がゼロショットで "できない" こと:
  学習していない物理スキルの実行
  （例: pick しか学習していないのに布を畳る、紐を結ぶ、蓋を回して開ける）
```

正確に言えば: **新しい意味解釈 × 既知の運動スキル** は可能だが、**新しい運動スキルの生成** は不可能。VLA は「物理スキルの辞書は固定で、VLM は辞書の引き方を広げているだけ」。

### 6.3 OTTER の重要な知見

OTTER が示した核心:

```
未見物体への実機ゼロショット成功率:
  OTTER:                62〜77%
  OpenVLA (fine-tuned):  0.6〜9%
  Octo (fine-tuned):     4〜12%
```

**VLM を fine-tune するとゼロショット能力がむしろ壊れる**。CLIP の視覚特徴を凍結（freeze）することで VLM の汎化能力を保存。

- fine-tune → 学習データの軌道に過適合 → ゼロショット能力消失
- freeze → VLM の汎化が保たれる → ゼロショットが効く
- **ただし freeze すると学習データ上の精度は下がる** → トレードオフ

### 6.4 ゼロショットの種類別まとめ

| ゼロショットの種類 | 現状 |
|-------------------|------|
| 新しい意味解釈（既知スキル内） | RT-2 以降、VLM 知識で可能 |
| 新しい環境・背景 | π0.5, GR-2 で 70〜94%。最も成功している軸 |
| 新しい物体 | OTTER で 62〜77%。VLM を freeze する必要あり |
| 新しいロボット（cross-embodiment） | **ほぼ未解決**。成功例は Mirage の視覚トリックのみ |
| 新しい物理スキル | **不可能**。成功例なし |
| Sim-to-Real | 限定的。50〜62.5% 程度 |
